# windPredictor

A sailing forecast system for local lake conditions.  A Random Forest model
trained on weather-station history predicts whether the sailing window will be
good.  NWP data from Open-Meteo enriches the output with expected wind/gust/
direction-consistency values.  The result is a self-contained `index.html`
deployed to GitHub Pages on every scheduled run.

---

## Architecture

```
input/
  scraper.py        Download daily xlsx files from Ecowitt weather station
  stitcher.py       Merge xlsx → data.parquet time-series
  open_meteo.py     Fetch NWP forecast (Open-Meteo, no API key needed)

model/
  features.py       Extract 28 features + compute daily sailing-quality targets
  train.py          Train Random Forest → model/weights.joblib
  predict.py        predict_now() — predict from current time; merge into rolling predictions.json
  history.py        Persist direct ML predictions + observed outcomes to predictions.db

render_html.py      Render self-contained index.html from predictions.json + predictions.db
deploy.py           Local pipeline: download → stitch → predict → render → git push
```

---

## How a prediction run works

`python model/predict.py` (called by GitHub Actions and `deploy.py`):

1. Runs `predict_now()` at the current hour.
2. The direct ML prediction targets **today** (if before window_start 08:00) or
   **tomorrow** (if after window_start).
3. Extended forecasts for the remaining days up to **day+3** are generated by
   scaling the direct probability with decay factors (×1.0, ×0.82, ×0.64 for
   days +1, +2, +3 beyond the direct target).
4. Results are **merged** with the existing `predictions.json` (rolling 7-day
   window, deduplicated by snapshot time + predicting date).
5. Direct predictions (not extended) are written to `predictions.db` for
   accuracy tracking.
6. `render_html.py` then reads `predictions.json` and `predictions.db` to
   produce `index.html`.

## Scheduling (GitHub Actions)

Six runs per day via `.github/workflows/forecast.yml`:

| Cron (UTC) | CET  | CEST |
|-----------|------|------|
| `0 4 * * *`  | 05:00 | 06:00 |
| `0 6 * * *`  | 07:00 | 08:00 |
| `0 9 * * *`  | 10:00 | 11:00 |
| `0 12 * * *` | 13:00 | 14:00 |
| `0 17 * * *` | 18:00 | 19:00 |
| `0 21 * * *` | 22:00 | 23:00 |

## Configuration (`config.toml`)

| Section | Key | Description |
|---------|-----|-------------|
| `[sailing]` | `window_start/end` | Sailing window times (local) |
| `[sailing]` | `wind_speed_min/max` | Speed thresholds in knots |
| `[sailing]` | `wind_dir_consistency_max` | Max circular std of direction (°) |
| `[sailing]` | `min_temperature` | Minimum air temp (°C) to count as "good" |
| `[prediction]` | `min_good_fraction` | Fraction of window that must be good (default 0.30) |
| `[location]` | `lat`, `lon` | Enables Open-Meteo NWP enrichment |
| `[model]` | `n_estimators`, `max_depth` | Random Forest hyperparameters |

## Local development

```bash
# Full pipeline (download 2 days, predict, render, push)
python deploy.py

# Skip download if xlsx files are already present
python deploy.py --no-download

# Predict and render only (use existing parquet)
python model/predict.py
python render_html.py

# Train model from scratch (needed on first run)
python model/train.py

# Dry-run (skip git push)
python deploy.py --dry-run
```

### First-time setup
1. Configure Ecowitt credentials in `input/scraper.py` (`DEVICE_ID`, `AUTHORIZE`).
2. Optionally enable NWP in `config.toml` by uncommenting `[location]` and
   adding `lat`/`lon`.
3. Run `python deploy.py --days 90` to bootstrap 90 days of history and train
   the model.

## Key design decisions

- **No snapshot list in config** — each workflow run predicts at its current
  time.  Multiple runs per day accumulate in `predictions.json`, building the
  probability-trend sparkline naturally.
- **Extended forecasts are display-only** — day+2/+3 entries are not stored
  in `predictions.db` to keep accuracy metrics clean.
- **Rolling 7-day `predictions.json`** — old entries are pruned automatically,
  so the file stays small and the UI always shows the last 7 days.
- **`predictions.db` is committed to the repo** — history survives across
  GitHub Actions runs without extra infrastructure.
